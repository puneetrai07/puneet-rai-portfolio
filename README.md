## 🏗️ Highlight: Funds Reservation System

Designed and implemented a system to prevent double-spending in a high-throughput financial environment.

### Key Concepts:

* Pessimistic locking (`SELECT FOR UPDATE`)
* Idempotency for safe retries
* Async settlement using message queues (IBM MQ)
* TTL-based reservation expiry

### Scale:

* 10M+ financial events/year
* 120–150 TPS
* <50ms lock hold time
